## Notes

- **Custom Domain (eleusisfx.uk)**: This is the primary production domain and should remain whitelisted at all times.
- **Vercel URLs**: These are historical production deployment URLs from Vercel. Include recent successful deployments to ensure continuity during rollbacks.
- **Localhost**: Required for local development. Only needed in Development environment in Supabase if configured separately.
- **Auth Callback Route**: All URLs point to `/auth/callback`, which is the Supabase OAuth callback handler configured in the Next.js app at `app/auth/callback/route.ts`.

## How to Apply

1. Log into your Supabase dashboard
2. Navigate to Project Settings → Auth → URL Configuration
3. Add each URL above to the "Authorized redirect URLs" section
4. Save changes
5. Test OAuth flow to verify URLs are properly whitelisted
